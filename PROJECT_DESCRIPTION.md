# Опис проекту Yallery

Yallery — це соціальна платформа для генерації та обміну AI-контентом (зображення та відео) з гейміфікацією та системою нагород.

## Архітектура проекту

Проект складається з трьох основних компонентів:

1. **y-allery-backend** — Backend API на NestJS (TypeScript)
   - REST API для всіх операцій
   - WebSocket для real-time повідомлень
   - TypeORM + MySQL для зберігання даних
   - Redis для кешування
   - Swagger документація

2. **y-allery-client** — Web адмін-панель на React
   - Управління контентом, користувачами, конкурсами
   - Аналітика та метрики
   - Налаштування AI-сервісів

3. **yallery_app** — Мобільний додаток на Flutter
   - Підтримка iOS, Android, Web
   - Генерація контенту
   - Соціальні функції

## Основні функції та фічі

### 1. AI-генерація контенту

#### Генерація зображень:
- **Моделі**: Aura Flow, Flux, Realistic Vision, Flux Pro Fine-tune, Ideogram
- **Параметри**: орієнтація (горизонтальна/вертикальна), стиль, кольори, розміри
- **Пакетна генерація**: 1-10 зображень за один запит
- **Редагування**: Bytedance Edit для редагування існуючих зображень
- **Автоматичний вибір тегів**: система автоматично підбирає теги для згенерованого контенту
- **Інтеграція з конкурсами**: можливість створювати пости для конкурсів

#### Генерація відео:
- **Моделі**: Bytedance та інші AI-сервіси
- **Генерація з зображення**: створення відео на основі зображення + промпт
- **Асинхронна обробка**: через черги для оптимізації

#### Система вартості:
- Вартість зберігається в таблиці `ai_settings`
- Різна вартість для різних моделей
- Відео: фіксована вартість 100 YEPs

### 2. Система нагород (Rewards System)

#### Типи нагород:

**Щоденні (один раз на день):**
- `DAILY_LOGIN` — щоденний вхід (10 YEPs)
- `POST_VIDEO_REWARD` — публікація відео (50 YEPs)
- `POST_PHOTO_REWARD` — публікація фото (30 YEPs)

**Одноразові:**
- `REGISTRATION_REWARD` — реєстрація (автоматично, 0 YEPs)
- `CONTEST_PARTICIPATION` — участь у конкурсі (100 YEPs)
- `RATE_APP` — оцінка додатку (100 YEPs)

**Автоматичні:**
- `LIKE_EARN` — за лайк вашого поста (5 YEPs)
- `LIKE_SPEND` — витрата на лайк (15 YEPs)
- `SHARE_REWARD` — реферальна нагорода (500 YEPs обом користувачам)
- `SHARE_YEPS` — шаринг поста (5 YEPs, раз на день)
- `CONTEST_WIN` — перемога в конкурсі (динамічна сума)
- `TOP_POST_REWARD_AUTHOR` — топ-пост дня (100 YEPs)
- `TOP_POST_REWARD_LIKER` — нагорода лайкерам топ-поста

#### Механіка:
- **YEPs (очки)** як внутрішня валюта
- **Історія транзакцій** через систему активностей
- **Клейм нагород** з перевіркою eligibility
- **Автоматичні cron jobs** для топ-постів

### 3. Соціальні функції

#### Пости:
- Публікація зображень та відео
- Лайки (з витратою YEPs)
- Коментарі
- Шаринг
- Теги та категорії
- Модерація (блокування, відхилення)

#### Профілі:
- Галерея контенту
- Статистика (зароблено/витрачено YEPs)
- Реферальна система
- Налаштування профілю

### 4. Конкурси (Contests)

#### Функціонал:
- Створення конкурсів адміністраторами
- Теми та промпти
- Участь через публікацію постів
- Вибір переможців адмінами
- Нагороди переможцям
- Інтеграція з Twitter (перевірка ретвітів для FINE_TUNE)

#### Типи:
- Звичайні конкурси
- FINE_TUNE (з перевіркою ретвітів)

### 5. Адмін-панель

#### Управління:
- Конкурси (створення, редагування, вибір переможців)
- Користувачі (блокування, модерація)
- Пости (блокування, відхилення, перегляд)
- Теги та стилі
- Налаштування AI-сервісів
- Система нагород
- Партнерства
- Розсилка повідомлень

#### Аналітика:
- Метрики користувачів
- Статистика генерацій
- Популярні пости
- Транзакції та активності

### 6. Аутентифікація та безпека

#### Методи входу:
- Email/Password
- Google OAuth
- Apple Sign In
- Twitter OAuth
- Telegram Login

#### Безпека:
- JWT токени (access + refresh)
- Role-based access control (Admin/User)
- Rate limiting
- WebSocket authentication

### 7. Платежі та монетизація

#### Пакети YEPs:
- 5000 YEPs
- 15000 YEPs
- 30000 YEPs

#### Інтеграція:
- Webhook обробка платежів
- Адаптивна система підписок (Adapty)

### 8. Сповіщення

#### Типи:
- Push-нотифікації (Firebase)
- In-app повідомлення
- WebSocket real-time оновлення
- Email-повідомлення (верифікація, скидання пароля)

#### Події:
- Нові лайки
- Коментарі
- Нагороди
- Завершення генерацій
- Перемоги в конкурсах

### 9. Технічні особливості

#### Backend:
- Асинхронна обробка через черги (Bull)
- Обработка зображень (завантаження, оптимізація)
- Інтеграція з AI-сервісами (fal.ai, OpenAI)
- Управління токенами AI-сервісів
- Міграції бази даних (TypeORM)
- Логування та моніторинг (Sentry)

#### Frontend (React):
- Redux для state management
- Захищені маршрути
- Responsive дизайн

#### Mobile (Flutter):
- BLoC архітектура
- Локалізація (багатомовність)
- Кешування контенту
- Offline режим
- Deep linking

## Технологічний стек

### Backend:
- **Framework**: NestJS, TypeScript
- **Database**: TypeORM, MySQL
- **Cache**: Redis
- **Real-time**: Socket.io (WebSocket)
- **Queues**: Bull
- **Notifications**: Firebase (push-нотифікації)
- **Documentation**: Swagger

### Frontend:
- **Framework**: React
- **State Management**: Redux
- **Styling**: Tailwind CSS

### Mobile:
- **Framework**: Flutter/Dart
- **Architecture**: BLoC
- **Services**: Firebase, Adapty (монетизація)

### AI Integration:
- **Services**: fal.ai, OpenAI, Bytedance SeedEdit

## Структура бази даних

### Основні таблиці:
- `users` — користувачі
- `posts` — пости (зображення/відео)
- `rewards` — типи нагород
- `user_rewards` — нагороди користувачів
- `activity` — історія активностей
- `contests` — конкурси
- `likes` — лайки
- `tags` — теги
- `ai_settings` — налаштування AI-сервісів
- `payments` — платежі
- `notifications` — сповіщення

## Висновок

Yallery — це комплексна соціальна платформа для AI-контенту з гейміфікацією, конкурсами та системою нагород, що стимулює користувачів до створення та обміну контентом.
